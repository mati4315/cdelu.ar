=== Endpoints para Gestión de Comunidad (Comunicaciones) ===

Prefijo Base: /api/v1

1. Listar Comunicaciones (Comentarios/Posts)
   - Método: GET
   - Endpoint: /api/v1/com
   - Parámetros de Query (Opcionales, pero soportados para paginación):
     - page: Número de página (ej. 1)
     - limit: Cantidad de resultados por página (ej. 10)
     - sort: Campo por el cual ordenar (ej. "created_at")
     - order: Dirección ("desc" para nuevos primeros, "asc" para antiguos)
   - Respuesta (200 OK):
     {
       "data": [
         {
           "id": 1,
           "titulo": "Título de la entrada",
           "descripcion": "Contenido del comentario",
           "image_url": "url_imagen.jpg",
           "video_url": "url_video.mp4",
           "user_id": 123,
           "created_at": "2026-03-24T10:00:00Z",
           "updated_at": "2026-03-24T10:00:00Z"
         }
       ],
       "pagination": { "page": 1, "limit": 10, "total": 50, "pages": 5 }
     }

2. Crear Comunicación
   - Método: POST
   - Endpoint: /api/v1/com
   - Headers: 
     - Authorization: Bearer <AUTH_TOKEN>
     - Content-Type: multipart/form-data
   - Body (form-data):
     - titulo: String (requerido)
     - descripcion: String (requerido)
     - image: File (opcional)
     - video: File (opcional)
   - Respuesta (201 Created): Objeto de la comunicación creada.

3. Obtener Comunicación por ID
   - Método: GET
   - Endpoint: /api/v1/com/:id
   - Headers: Authorization: Bearer <AUTH_TOKEN>
   - Respuesta (200 OK): Objeto de la comunicación detallada.

4. Editar Comunicación
   - Método: PUT
   - Endpoint: /api/v1/com/:id
   - Headers: 
     - Authorization: Bearer <AUTH_TOKEN>
     - Content-Type: application/json
   - Body JSON:
     {
       "titulo": "Nuevo título",
       "descripcion": "Nuevo comentario modificado"
     }
   - Respuesta (200 OK): Objeto de la comunicación modificado.

5. Eliminar Comunicación
   - Método: DELETE
   - Endpoint: /api/v1/com/:id
   - Headers: Authorization: Bearer <AUTH_TOKEN>
   - Respuesta (204 No Content): Eliminación exitosa.

=== Notas Técnicas ===
* Autenticación: Se requiere authToken del localStorage y usar Authorization: Bearer <TOKEN>.
* Notificaciones: Manejar con la función showToast() para mostrar alertas.
* Para paginación usar los controles en el frontend y cambiar los parámetros query '?page=X&limit=Y'.
